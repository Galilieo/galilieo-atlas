## Variant: Archive / 活体档案

### Design stance

以研究档案和设计标本的方式讲项目，把产品世界与技术系统并置。

### Key choices

- Layout: 主标本区 + 右侧能力档案
- Typography: 档案式 mono 标签 + 编辑衬线命题
- Interaction: World / System 切换，内容行微位移，方形 CTA 旋转

### Trade-offs

- Strong at: 信息表达、个人开发者身份、可扩展性
- Weak at: 信息密度最高，情绪氛围不如 Aperture

### Best for

- 想同时展示产品思考与工程能力
