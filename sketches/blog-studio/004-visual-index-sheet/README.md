## Variant: 选择中心与图像索引

### Design stance

设置主界面只展示当前状态，点“更改”进入底部选择面板；移动博客用两列图片索引提高扫描密度。

### Key choices

- Studio：分类、标签、封面都以摘要行呈现，通过 bottom sheet 选择；没有可创建新值的自由输入。
- 移动博客：两列、128px 图片、16px 最多三行标题、隐藏摘要、日期覆盖图片。
- 交互：三类 bottom sheet、标签搜索/多选、封面图库选择、Studio/博客视图切换。

### Trade-offs

- Strong at：设置页短而清楚；一屏可浏览约四篇文章；图片存在感强。
- Weak at：选择需要多一步；很长的文章标题在窄列中仍可能显得拥挤；不展示摘要。

### Best for

- 文章数量增长后强调快速扫描和图片浏览。
- 喜欢 XingHuiSama 的移动图库密度，但仍保留 Galilieo 自己的配色与内容。
