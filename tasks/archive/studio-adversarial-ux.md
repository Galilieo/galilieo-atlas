# Studio 对抗式 UX 审计

## 目标

用异常顺序、键盘、脏状态、长内容和窄视口攻击 Studio 的关键写作流程，只修会造成误操作、状态丢失或明显上下文错位的问题。

## 范围

- 文章设置、发布检查、新建草稿、Cover 面板。
- Category / Tags 的空值、重复、长值和键盘行为。
- 抽屉/弹窗的焦点、返回位置、层级和状态隔离。
- 桌面与 390px 移动视口。

## 不修改

- 公开站视觉、博客内容和服务端部署流程。
- 已通过验收的 Tags creatable combobox 结构。
- 无问题证据的样式和功能。

## 对抗矩阵

| 场景                       | 预期                                          |
| -------------------------- | --------------------------------------------- |
| 新建弹窗 Tab 到末尾        | 焦点留在弹窗内                                |
| 设置/发布抽屉 Tab 到末尾   | 焦点留在当前抽屉内                            |
| 关闭抽屉/弹窗              | 焦点回到原触发器                              |
| 取消新建后重新打开         | 标题、Slug、摘要、Category、Tags 都是干净状态 |
| 当前文章有未保存修改时新建 | 必须确认，不能静默切走                        |
| Tags 菜单首次 Esc          | 只关菜单；再次 Esc 才关容器                   |
| Settings → Cover → 关闭    | 返回 Cover 触发器，不跳到背景                 |
| 短高度/390px               | 无横向溢出，菜单不被 footer 裁切              |
| MDX 只读                   | 摘要与写入控件不可操作                        |

## 步骤

1. 真实浏览器复现并记录问题。
2. 冻结 blocking / high / medium 修复。
3. 先写失败契约，再实现最小修复。
4. 复测桌面、移动、键盘和状态往返。
5. 完整 verify、独立审查并归档。

## 验收

- [x] 对抗矩阵全部执行。
- [x] 高/中风险问题修复并有回归契约。
- [x] 不修改真实文章、不创建测试草稿。
- [x] 完整 verify、独立审查并归档。

## 结果

- 取消新建后残留标题与 Slug：已改为统一重置表单和独立 taxonomy 状态。
- 新建弹窗、设置抽屉和发布抽屉的焦点逃逸：已增加 dialog / aria-modal / inert、焦点陷阱和触发器返回。
- 当前文章 dirty 时直接进入新建流程：已增加明确确认，拒绝时保留当前状态。
- Tags Escape 被输入框永久吞掉：已改为首次关闭菜单、再次关闭抽屉。
- Cover 从设置打开并关闭：设置保持打开，焦点返回 Cover 触发器。
- 390×844：页面无横向溢出，抽屉 390px、Tags 字段 345px，候选菜单完整可见。
- 独立审查报告无 Blocking；已修复点击监听器把 `MouseEvent` 传给 options 函数的脆弱写法，以及 Tags 焦点真正离开 combobox 后菜单残留的问题。
- 最终浏览器复测确认 Tags 菜单失焦关闭、抽屉恢复 `inert` 且焦点返回摘要触发器。
- `node scripts/verify.mjs` 最终通过：Studio `76/76`，Astro check / build / 生产隔离全部通过。

## 命令

```bash
node scripts/verify.mjs
```

## 风险

- 焦点管理不能破坏 Tags 内部 Escape 两阶段关闭。
- 新建保护必须保留本地恢复副本，不应擅自丢弃当前文章。
- 多层浮层只允许 Cover 作为设置的从属面板。
