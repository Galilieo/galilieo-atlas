# Studio 可新增分类与标签

## 目标

允许在本地 Galilieo Studio 写作时新增 Category 和 Tags，同时保留已有值快速选择，不引入数据库、CMS 或独立 taxonomy 文件。

## 范围

- 文章设置侧边栏与新建草稿弹窗可新增并选中主分类。
- 标签底部面板可从搜索输入新增并选中标签。
- Studio API 接受符合约束的新分类/标签，并继续拒绝空值、过长值和控制字符。
- 新值只在文章保存或草稿创建时写入 frontmatter，之后由现有文章扫描自动进入候选列表。
- 同步 UI/server tests 与内容、设计指南。

## 不修改范围

- 不建立线上写接口、数据库、全局 taxonomy 管理页或删除/重命名迁移工具。
- 不改变封面、博客目录和公开路由。

## 验收

- Category 可新增、立即选中且保存后成为候选项。
- Tags 可新增、立即选中；相同名称不重复创建。
- 空白、控制字符和过长 taxonomy 被服务端拒绝。
- 当前文章与新建草稿状态隔离，MDX 继续只读。
- `node scripts/verify.mjs` 完整通过。
