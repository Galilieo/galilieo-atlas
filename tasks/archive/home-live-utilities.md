# 首页实时功能与音乐卡

## 目标

在不增加服务端的前提下，为首页接入真实 GitHub 活动、访客粗略环境信息、天气和网易云原生播放器。

## 范围

- GitHub 最近 30 天 × 7 类 Public Events 活动格
- 浏览器时区、IP 粗定位、上海回退与 Open-Meteo 当前天气
- 网易云歌单 `18145116776` 的静态清单与原生 `<audio>` 播放器
- 缓存、错误状态、Astro 页面切换清理和无 JavaScript 回退
- 桌面/移动、浅色/深色、Reduced Motion 验证

## 不修改范围

- 不增加后端、数据库、服务端 API 或客户端密钥
- 不复制 XingHuiSama 的 CC BY-NC 源码或素材
- 不伪造 GitHub Contributions、位置或天气
- 不修改 Heart Island SVG 和首页整体网格
- 第一版不增加歌词、音乐详情页、随机/单曲循环和音量面板

## 数据边界

- GitHub：官方 Public Events，仅最近 30 天，缓存 30 分钟
- IP 位置：城市级粗略位置，缓存 24 小时；失败回退上海
- 天气：Open-Meteo 当前天气，缓存 15 分钟
- 音乐：开发时同步静态元数据；运行时使用网易云稳定 outer URL

## 步骤

1. TDD 实现 GitHub 活动成功、缓存与失败状态。
2. TDD 实现访客时区、IP 粗定位、上海回退和天气。
3. 生成静态网易云清单并 TDD 实现播放、切歌、不可播放跳过与清理。
4. 收敛组件结构和响应式样式。
5. 更新文档和首页结构检查。
6. 运行完整自动验证与浏览器矩阵。

## 验收

- 网络失败不会留下永久加载状态，也不会显示虚构数据。
- 无 JavaScript 时保留 GitHub 外链、上海地点和可读音乐来源。
- 不自动播放；Reduced Motion 下封面不旋转。
- 页面往返不会重复请求、定时器或媒体监听器。
- `pnpm run verify`、首页检查与 `git diff --check` 通过。

## 风险

- GitHub 未认证 API 按 IP 限流。
- IP 地理服务为第三方粗略推断，可能错误或不可用。
- 网易云外链受版权和地区限制，当前 5 首中有 1 首不可站外播放。
