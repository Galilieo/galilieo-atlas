# 首页最终原型 v1

- 状态：已由 `codex-luminous-archive-v2` 的 v2.4 方案取代并归档（2026-07-11）
- 创建日期：2026-07-11
- 当前阶段：依据盘点
- 适用范围：Galilieo 首页视觉原型，不包含开发实施

## 目标

在修改首页代码之前，基于当前真实内容、现有 P0 首页、已经确认的日间与夜间高保真参考图，整理一套可直接指导后续开发的首页最终原型。

原型确认后，Codex、Hermes 或维护者应能明确判断首页各区块的布局、主题表现、响应式行为、素材用途和动效边界，不需要在开发过程中重新猜测设计方向。

## 已确认边界

### 本任务包含

- 完整桌面首页的信息层级与高保真原型。
- 同一结构下的日间和夜间正式主题。
- 平板和移动端的关键布局与内容优先级。
- Header、Hero、状态卡、项目、博客、归档、关于和 Footer 的统一组件语言。
- 默认、Hover、Focus、Active、禁用或无内容等必要状态。
- Reduced Motion、无 JavaScript 和素材缺失时的降级说明。
- 背景、项目封面和文章封面的占位及后续替换规则。

### 本任务不包含

- 修改 Astro、CSS、TypeScript 或客户端脚本。
- 制作或选定最终背景图片、头像、项目封面和文章封面。
- 设计项目详情、博客详情、归档、关于和 404 等其他页面。
- 引入 React、Vue、Svelte、Tailwind、GSAP、Three.js 或新的运行时依赖。
- 3D 岛屿、重型 Canvas、动态海面、复杂粒子、音频或页面转场。
- 在当前内容规模下加入没有必要的项目筛选或博客搜索。
- 虚构项目、文章、实习成果、统计数字、时间、天气或 GitHub 数据。

## 参考网站使用原则

允许参考 XingHuiSama 的日夜氛围、玻璃层次、背景分层、布局密度、响应式处理和动效方法；也允许阅读其开源代码以理解实现思路，最终仍以 Astro 和原生 CSS 适配 Galilieo 的现有结构。

不直接把对方的头像、背景图、插画、Logo、文案和品牌表达作为 Galilieo 生产内容。若后续需要直接复用具体代码或素材，必须先核对对应文件的许可证、署名要求和使用范围，并在实施任务中记录来源。

## 依据盘点结论（2026-07-11）

### 保留

- 保留当前 P0 的完整区块顺序、真实数据来源、路由、语义标题和渐进增强契约。
- 保留桌面 Hero 的“身份 → Heart Island → 当前状态”三层关系；平板状态面板下移，移动端按身份、岛屿、状态顺序单列。
- 保留 `IslandArtwork` 的主体几何、内部轮廓和水纹整体布局；原型只调整它所在场景的空间、光线、背景和说明层，不重画主体。
- 保留 Galilieo、AI 应用开发者 / 软件工程学生、三个真实项目、四篇当前公开文章、实习摘要和成长轨迹等可追溯内容。
- 保留 Instrument Serif / Noto Serif SC / Plus Jakarta Sans / IBM Plex Mono 的编辑感字体分工，以及 `--home-*` 的双主题语义。

### 借鉴

- 从 Galilieo 日夜参考图借鉴首屏“环境场景承载品牌、内容面板浮于场景之上”的层级，但不复制其中的完整插画资产。
- 借鉴参考图首屏信息紧凑、Quick Overview 与 Hero 连续、下方项目与博客形成主次双栏的仪表盘节奏。
- 从第三方参考只借鉴背景资源层、主题遮罩层、强弱面板层的分离，以及移动端单列、外围装饰收缩、正文优先的响应式原则。
- 日间使用暖骨白、雾蓝灰和清透水面感；夜间使用深海蓝黑、靛蓝灰和少量月光紫蓝。两者共享布局，不做简单反色。

### 替换或补足

- 当前 P0 首屏的中央场景边界清楚但环境叙事偏弱；原型补足天空、雾、海面、远山和光晕的抽象资源层，同时继续使用现有受保护岛屿素材。
- 当前首屏三栏彼此较独立；原型用一个连续场景和局部强面板重新建立整体构图，让身份和状态围绕 Heart Island，而不是三个并列产品卡。
- 当前 Quick Overview 与 Hero 之间联系较弱；原型让数据条和三张快捷卡成为首屏场景的下沿，而不是下一段孤立内容。
- 当前项目、博客、归档和关于区视觉语言一致但层级过于平均；原型明确“项目为主视觉卡、博客为编辑列表、归档为轻量账本、关于为收束叙事”的差异。
- 当前缺少真实封面时继续使用明确的抽象占位，不把参考图中的岛屿插画或 AI 生成图冒充项目封面。

### 禁止虚构或复制

- 不展示时间、天气、音乐播放器、GitHub 热力图、获赞数、虚构项目数、虚构文章数、头像或未确认成果。
- 不展示 Agent Playground、Recipe Hub 等不存在的项目；项目只使用 heart-island、AI Chat App、Galilieo Portfolio。
- 不复制第三方动漫背景、人物、播放器、悬浮工具、搜索框、精确色值、品牌和素材。
- 不把设计参考图导入生产源码，不把临时场景当作已经选定的最终背景资产。

## 原型制作规格

- 评审载体：`docs/assets/homepage-final-prototype-v1.html`，单文件、无构建依赖，不进入 Astro 路由。
- 视图覆盖：同一响应式实现覆盖桌面日间、桌面夜间和移动端；内置主题切换与移动菜单用于评审状态。
- 桌面画布：以 1440×1000 为主基线，最大内容宽度约 1360px；首屏保持一个视口内可理解，Quick Overview 紧接场景下沿。
- 平板画布：以 1024×768 检查 Hero 两列与状态下移，不新增相邻断点。
- 移动画布：以 390×844 检查单列顺序、触控尺寸、横向溢出和装饰收缩。
- 场景分层：页面底色 → 天空/海面抽象层 → 主题遮罩 → Heart Island → 身份/状态强面板 → 交互与文字。
- 面板分级：Hero 身份与当前状态使用强面板；Quick Cards、项目卡使用普通面板；博客、归档使用边线和留白，不把所有内容都做成厚重玻璃卡。
- 交互边界：只表现主题切换、移动菜单、链接 Hover/Focus、克制卡片位移和 Reduced Motion；不实现搜索、筛选、播放器或持续高成本动画。
- 素材状态：现有岛屿素材标记为“保留”；背景和封面均标记为“可替换资源层”，原型确认不等于最终素材确认。

## 原型验收标准

- 首页首先清楚表达 Galilieo 是谁、技术方向、真实项目与内容入口。
- Heart Island 是首屏视觉锚点，但不会遮挡身份、导航和行动入口。
- 日间与夜间具有共同品牌识别，不是简单反色。
- 页面氛围可以看出对参考站的设计研究，同时仍属于 Galilieo 的内容与视觉表达。
- 所有展示内容可追溯到当前代码、配置或 Content Collections。
- 移动端按内容优先级重排，不保留无价值装饰和拥挤并排面板。
- 背景、封面或 JavaScript 缺失时，主要内容依然自然、可读、可访问。
- 原型清楚到足以另行编写开发计划，不需要实施者自行决定关键视觉问题。

## 开发门禁

在用户明确确认完整首页原型之前：

- 不修改首页业务代码和样式。
- 不创建首页开发实施计划。
- 不把临时原型素材接入生产页面。
- 不扩展到其他页面的视觉重构。

原型确认后，后续开发任务才可以拆分组件修改、素材接入、浏览器验证和质量检查步骤。
