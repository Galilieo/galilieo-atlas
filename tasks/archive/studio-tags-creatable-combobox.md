# Studio Tags 原位可创建多选

## 目标

让 Tags 的新增能力可发现，并让选择菜单保持在触发字段附近，不再使用远离上下文的底部面板。

## 范围

- 当前文章设置抽屉与新建草稿弹窗的 Tags 控件。
- Tags 搜索、新增、选择、删除、键盘和菜单定位。
- UI 契约、内容指南与设计指南。

## 不修改

- Category 单选与新增流程。
- 封面底部选择面板。
- 服务端 Markdown/API 边界与公开站页面。

## 步骤

1. 研究 WAI-ARIA、Atlassian Select 与 React Select Creatable。
2. 复现旧底部面板的位置和可发现性问题。
3. 先写失败契约，再实现原位 creatable multi-select。
4. 验证当前文章、新建草稿、状态隔离和窄高视口菜单翻转。
5. 更新指南，运行完整 verify 和独立审查。

## 验收

- [x] 已选值、搜索输入与新增动作位于同一字段。
- [x] 输入不存在值时首行显示“创建”，Enter 立即选中。
- [x] 顶部摘要打开设置并聚焦 Tags，不再打开底部面板。
- [x] 菜单空间不足时自动向上展开，不被 footer 裁切。
- [x] 当前文章与新建草稿状态隔离，未操作保存时不写 Markdown。
- [x] 完整 verify、独立审查并归档。

## 命令

```bash
node scripts/verify.mjs
```

## 风险

- 弹窗/抽屉滚动容器可能裁切绝对定位菜单；通过运行时可用空间计算决定上下方向和最大高度。
- 动态选项必须继续使用 HTML 转义，并保留至少一个当前文章标签的保护。
