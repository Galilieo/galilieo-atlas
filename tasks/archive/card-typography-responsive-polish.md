# Web / Mobile 卡片与字体响应式优化

## 目标

在不改变 Galilieo Atlas 既定结构和视觉方向的前提下，分别校准桌面、平板和移动端的卡片尺寸、图片占比、字号、行高、内边距与信息密度；同时提升本地 Studio 的文章库、写作区和设置控件可读性。

## 范围

- 首页：个人档案卡、GitHub / 音乐卡、主要项目与精选文章卡。
- 博客目录：桌面两列、581–760px 单列、≤580px 双列图片索引。
- 项目目录：桌面两/三列与移动单列项目卡。
- Studio：文章库条目、当前文章信息、正文编辑字号、摘要 chips、设置抽屉中的 Category / Tags / Cover 控件。
- 为关键响应式尺度补充结构契约，并同步设计维护说明。

## 不修改范围

- 不改变首页 60:40 主次网格、内容顺序、背景图、岛屿与水纹几何。
- 不改变博客 ≤580px 双列、581–760px 单列方向。
- 不改变真实文案、Content Collections、路由、构建方式或部署。
- 不新增框架、依赖、动画运行时或全局 UI 组件。
- 不修改详情页正文排版，除非浏览器审计发现本轮样式泄漏。

## 基线事实

- 首页 1440：Profile `718×494`；第二行外卡约 `718×456 / 478×456`；项目/文章内卡约 `660×264 / 420×269`；meta `9px`、标签 `8px`、摘要 `12px`。
- 首页 390：外卡宽 `354px`；内容内卡 `318×222`；meta `9px`、标签 `8px`、摘要 `12px`；个人摘要 `12px`。
- 博客 1440：文章卡 `424×509`，图片仅 `136px`，标题 `28.8px`；390：双列卡 `176×208`，图片 `100px`，标题 `16px`，meta `9px`。
- 项目 1024：卡 `464×466`、图片 `154px`；390：卡 `362×332`、图片 `110px`，meta/tag/footer `9px`。
- Studio：文章条目高 `53.8px`，标题 `11px`、meta/status `8px`；移动正文 `13px`、摘要 chip `9px`、设置字段约 `10–11px`。

## 冻结尺度

1. 首页内容卡：保留外卡与 60:40 比例；功能性 meta 提升到 `10px`，标签到 `10px`，摘要到 `13px`；文章标题上限收敛，移动摘要不低于 `13px`。
2. 博客卡：完整卡图片提升至约 `160px`；缩减正文区无效留白；标题使用适合卡宽的 `24–28px`。≤580px 图片改为 `16:9`，390 基本保持当前高度，580 自动获得更完整封面；标题 `17px`、meta `10px`。
3. 项目卡：桌面图片使用约 `16:7` 比例，移动约 `5:2`；移动标题 `22px`、摘要 `13px`、meta/tag/footer `10px`。
4. Studio：文章条目约 `60–64px`，标题 `12px`、meta / 状态 `10px`；正文桌面/移动不低于 `15px`；设置字段主体 `12–13px`、提示 `10px`，触控高度至少 `42px`。
5. 不用 `overflow-x: hidden` 掩盖问题；390px 必须满足 `scrollWidth <= innerWidth`。

## 步骤

1. 为响应式尺度补充最小契约，确认 RED。
2. 只修改现有 CSS 选择器和断点，不重排组件。
3. 在 1440×1000、1024×768、760×900、580×844、390×844 实测卡片几何和字体。
4. 检查浅色、深色、200% 文本缩放、Reduced Motion、无 JavaScript与控制台。
5. 运行完整 verify，并由不同实例做独立视觉/代码审查。

## 验收

- 桌面卡片不因长标题产生失控高度，图片区不再像窄横条。
- 390px 博客仍是双列，图片与标题比例自然；580px 不再沿用固定 100px 的过扁封面。
- 项目卡图片在桌面与移动端均具有明确视觉份量，元信息不低于可读下限。
- Studio 常用文字不再集中在 `8–11px`，但仍保持写作优先和紧凑工具感。
- 所有目标视口无横向溢出、裁切或组件互相覆盖。
- `pnpm run verify` 完整通过。

## 风险

- 字号提升可能增加长标题高度；通过行数限制和卡宽实测控制，而非缩回不可读字号。
- 580px 双列图片增高会增加列表长度，但比固定 100px 的过扁封面更符合图片索引定位。
- Studio 字号提升可能压缩窄侧栏；优先增加条目高度和合理截断，不扩大桌面固定栏到侵占编辑区。

## 结果

- 首页桌面首行由约 `494px` 收敛为约 `458px`；GitHub 卡约 `244px`，功能文字提升至 `9–12px`。首页内容卡 meta / 标签 / 摘要分别提升至约 `10px / 10px / 13px`。
- 博客完整卡封面提升至 `160px`；390px 双列卡约 `176×212px`、封面约 `98px`、标题 `17px`、meta `10px`；580px 封面按 `16:9` 增至约 `151px`。
- 项目移动卡约 `362×374px`、封面 `144px`、标题 `22px`、摘要 `13px`；独立视觉审查无 Blocking / Medium。
- Studio 文章条目约 `60px`，标题 `12px`、meta / 状态 `10px`，编辑正文 `15px`，设置字段 `12–13px`。
- 1440、1024、760、580、390 及 720 CSS px 等效 200% 页面缩放宽度均无横向溢出；浅色、深色、Reduced Motion 和无 JavaScript 回退通过。
- `node scripts/verify.mjs` 最终通过：Astro check `0` errors / warnings / hints，构建 `20` 页，Studio `76/76`，生产隔离通过。
- 最终 sitemap 爬取覆盖 `19` 页、`37` 个站内目标、`9` 条 RSS，HTTP / SEO 失败均为 `0`。
