# 全局音乐、回顶与博客封面

## 目标

让网易云播放器在 Astro 页面之间持续播放，提供左下紧凑全局面板和右下回顶入口，并为博客卡补充安全的二次元风 Demo 封面。

## 范围

- `BaseLayout` 全局持久播放器
- 首页音乐卡作为同一音频状态的同步视图
- 全局回到顶部控件
- 四篇博客的许可明确或原创占位封面
- 桌面、移动、浅色、深色、Reduced Motion、无 JS 与页面往返验证

## 不修改

- 不改变首页两行仪表盘结构
- 不增加第二个音频实例或自动播放
- 不引入框架、播放器依赖或后端
- 不修改 Heart Island SVG、岛屿轮廓和水纹
- 不使用来源或许可不明的第三方动漫图片

## 实施

1. 抽取全局音乐组件与唯一 `<audio>`。
2. 建立事件驱动状态同步，首页卡只做控制视图。
3. 加入左下唱片球、紧凑歌单面板及移动端底部面板。
4. 加入右下回顶锚点与滚动可见性增强。
5. 为博客生成或引入许可明确的 Demo 封面并填写 `cover`。
6. 完成 TDD、生命周期、视觉和全量构建验证。

## 验收

- 播放中从首页进入项目/博客页面不中断、不重置进度。
- 全站只有一个 `<audio>`；首页与悬浮面板曲目和播放状态一致。
- 面板支持关闭、Esc、曲目切换、进度与网易云外链。
- 回顶无 JS 时仍为 `#top` 原生链接；Reduced Motion 不平滑滚动。
- 移动端控件不遮挡主要导航且无横向溢出。
- 博客封面均有明确安全来源，卡片文字保持可读。
- `pnpm run verify` 与 `git diff --check` 通过。

## 风险

- 网易云外链可能因版权或地区变化失效，必须保留跳转与单曲错误状态。
- 全局音频节点必须脱离每页 cleanup，并通过 Astro persist 保留。
- 第三方图片若许可无法核实则不得进入仓库。
